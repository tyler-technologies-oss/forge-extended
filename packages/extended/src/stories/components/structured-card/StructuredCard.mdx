The Structured Card component provides a pre-configured card layout optimized for the most common card patterns in Tyler Cloud applications. Built on top of the Content Scaffold component, it combines a card shell with a structured header, body, and footer layout—providing designated slots for titles, actions, content, and buttons.

This component is the recommended approach for building card-based UIs throughout Tyler Cloud applications. It ensures visual consistency and eliminates the repetitive work of manually configuring the same card structure for common patterns like data tables, detail panels, workflow cards, and form containers.

## Usage

The Structured Card is the recommended way to build consistent card-based UIs throughout Tyler Cloud applications. It provides a standardized layout pattern that ensures visual consistency while eliminating the need to manually configure the same card structure repeatedly.

## Structure

The component provides several slots for organizing content:

- **before-title** - Optional slot for content before the title in the header (e.g., back button, icon)
- **title** - Title/heading content for the card
- **header-actions** - Optional actions or controls displayed in the header (e.g., menus, icon buttons)
- **body** - Main content area that expands to fill available space
- **footer-start** - Optional content at the start of the footer (e.g., pagination controls)
- **footer-secondary-action** - Optional secondary action button (typically cancel or back)
- **footer-primary-action** - Optional primary action button (typically save or submit)

> **Note:** For the Structured Card to properly fill available space, ensure its parent container has a defined height.

## Examples

### Data Table with Pagination

Use the `with-table` attribute when the body contains a table. This attribute:

- Removes the default body padding to allow tables to extend edge-to-edge
- Sets the gap between sections to `0` for a seamless table integration
- Removes the footer's bottom padding for consistent spacing

This ensures tables fit naturally within the card while maintaining proper alignment with the header and footer elements.

## Layout Details

The Structured Card automatically handles spacing and layout:

- The header includes a bottom border to visually separate it from the body content
- When the `before-title` slot is empty, the title automatically receives left margin for proper spacing
- The footer provides separate areas for secondary and primary actions with appropriate spacing
- All spacing uses Forge design tokens to ensure consistency across the application

## CSS Custom Properties

- `--forge-structured-card-body-height` - Controls the height of the body content. Defaults to `auto`. Set this to a fixed value to enable scrollable content.
