The Content Scaffold provides a structured layout for organizing content into distinct header, body, and footer sections. It uses CSS Grid to create a flexible container where the header and footer automatically size to their content, while the body expands to fill all available space.

This component helps standardize the layout of cards, dialogs, side panels, and any UI pattern requiring a consistent header/body/footer structure. By providing a reusable layout primitive, it ensures visual consistency across your application while eliminating the need to recreate common layout patterns.

## Usage

The Content Scaffold is ideal for creating consistent content layouts where you need a fixed header and/or footer with a scrollable body section. The component automatically handles the layout arrangement using CSS Grid, ensuring the body section takes up all remaining vertical space.

## Structure

The Content Scaffold supports two layout modes for headers and footers, automatically determined by which slots you use:

### Full-Width Mode (Recommended for Complex Layouts)

When you place content in the `header` or `footer` slots, full-width mode is automatically enabled, giving you complete control over the layout:

- **header** - Full-width header content slot (hides the multi-slot header layout when used)
- **body** - Main content area that expands to fill available space (1fr)
- **footer** - Full-width footer content slot (hides the multi-slot footer layout when used)

### Standard Mode (Built-in Layout)

When the `header` and `footer` slots are empty, the component provides a built-in toolbar layout with multiple slots:

- **before-header-start** - Content displayed before the header start slot (e.g., back button, icon)
- **header-start** - Content displayed at the start of the header section (e.g., title, heading)
- **header-end** - Content displayed at the end of the header section (e.g., actions, menus)
- **body** - Main content area that expands to fill available space (1fr)
- **footer-start** - Content displayed at the start of the footer section (e.g., pagination)
- **footer-end** - Content displayed at the end of the footer section (e.g., action buttons)

All slots are optional, allowing you to use only the sections you need for your specific use case.

## Common Use Cases

The Content Scaffold is particularly useful for:

- Detail panels with a title header and action footer
- Card layouts with header toolbars and footer actions
- Content areas that need a consistent three-section layout
- Scrollable content regions with fixed headers and footers

## Layout Behavior

The component uses `display: grid` with `grid-template-rows: auto 1fr auto` to arrange the three sections:

- Header and footer use `auto` sizing, taking only the space their content requires
- Body uses `1fr`, expanding to fill all remaining vertical space
- The container is set to `height: 100%`, inheriting height from its parent

> **Note:** For the Content Scaffold to properly fill available space, ensure its parent container has a defined height.

## CSS Custom Properties

The component exposes several CSS custom properties for customization:

- `--forge-content-scaffold-header-height` - Controls the height of the header section. Defaults to `auto`.
- `--forge-content-scaffold-header-background` - Controls the background color of the header. Defaults to `transparent`.
- `--forge-content-scaffold-header-min-height` - Controls the minimum height of the header. Defaults to `48px`.
- `--forge-content-scaffold-body-height` - Controls the height of the body section. Defaults to `auto`.
- `--forge-content-scaffold-body-padding-inline` - Controls the inline padding applied to the body section. Defaults to Forge's medium spacing token.
- `--forge-content-scaffold-footer-height` - Controls the height of the footer section. Defaults to `auto`.
- `--forge-content-scaffold-footer-background` - Controls the background color of the footer. Defaults to `transparent`.
- `--forge-content-scaffold-footer-min-height` - Controls the minimum height of the footer. Defaults to `48px`.
- `--forge-content-scaffold-footer-full-padding` - Controls the padding applied to full-width footer content. Defaults to Forge's small spacing token.
